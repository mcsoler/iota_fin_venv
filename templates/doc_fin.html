Backend

El backend, en el contexto del desarrollo web y software, es la capa de procesamiento de datos y lógica detrás de una aplicación o sitio web. Técnicamente, comprende los servidores, bases de datos, y aplicaciones que corren en el servidor, que juntos procesan, almacenan y gestionan la información.

Lenguaje de Programación:

Python 3.x.

Framework Web:

Flask, para manejar las solicitudes HTTP, la lógica del servidor, y las respuestas.

Gestión de Dependencias:

Pip3 para la gestión de librerías externas.

Servidor de Producción:

Gunicorn y uWSGI para desplegar la aplicación en producción.

Frontend

El frontend es la parte del desarrollo web que se dedica a la parte frontal de un sitio web, en pocas palabras del diseño de un sitio web, desde la estructura del sitio hasta los estilos como colores, fondos, tamaños hasta llegar a las animaciones y efectos

Interfaz de usuario:

HTML5, CSS3, y JavaScript

Biblioteca de Frontend:

Bootstrap y JavaScript para el diseño responsivo.

Comunicación con Backend:

Framework Flask con python-redis

Bases de datos

Recopilación organizada de información o datos estructurados, que normalmente se almacena de forma electrónica en un sistema informático. Normalmente, una base de datos está controlada por un sistema de gestión de bases de datos (DBMS)

Sistema de Base de Datos:

Redis, utilizado para almacenar datos en memoria con alta velocidad de acceso, como sesiones de usuario, caché, o estados temporales de la aplicación.

Cifrado de Datos

El cifrado en ciberseguridad es la conversión de datos de un formato legible a un formato codificado. Los datos cifrados solo se pueden leer o procesar luego de descifrarlos.

Biblioteca de Criptografía:

Cryptography.fernet para cifrar archivos y datos antes de su transmisión

Cifrado IOTA_SDK:

Ed25519 es un algoritmo de firma digital ampliamente adoptado conocido por su fuerte seguridad y eficiencia. Firmar con Ed25519 garantiza datos integridad, autenticidad y no repudio. Al firmar con Ed25519, una clave privada genera una firma digital que se puede verificar con la clave pública correspondiente.

Claves de Cifrado:

Claves simétricas generadas y manejadas de manera segura para el uso de IOTA_SDK y Fernet

Integración con Tangle

A diferencia de la cadena de bloques tradicional, Tangle es un sistema de contabilidad distribuida que utiliza una estructura de datos conocida como Directed Acyclic Graph (DAG) para almacenar sus transacciones.

SDK de IOTA:

Biblioteca iota_sdk, utilizado para interactuar con la red Tangle

Transacciones Tangle:

Capacidad de enviar datos y archivos cifrados como mensajes o transacciones a un nodo de Tangle específico.

Seguridad de Transacciones:

Aseguramiento de la integridad y la no alteración de los datos durante la transmisión.

Deployment

En el contexto de la tecnología de la información y el desarrollo de software se refiere al proceso de poner en funcionamiento una aplicación o un conjunto de aplicaciones en un entorno de producción.

Contenedores:

Docker para empaquetar la aplicación y sus dependencias.

Orquestación:

Kubernetes o Docker Swarm si se requiere escalar y gestionar múltiples contenedores
